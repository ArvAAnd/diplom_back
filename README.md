# AskExpert Backend

The server-side service for the AskExpert platform. It exposes a RESTful API to manage user registration, authentication tokens, rating calculations, and a matchmaking system based on relational data.

## Tech Stack
* Python
* Flask (Web Framework)
* SQLite (Relational Database)
* SQL

## Database Architecture
The relational database is managed via SQLite (`sqlite3`) and contains 6 structured entities:
* Users: Stores authentication data, contact details, and current average rating.
* Themes: Stores unique system categories and titles.
* User Interested Themes / User Expert Themes: Intermediary tables handling many-to-many relationships between users and their skills/interests.
* Tokens: Manages active user authorization safe tokens generated by Python's `secrets` module.
* Rating Table: Keeps records of unique rating scores given between participants to prevent duplicate entries.

## Core API & Functionality
* REST Routing: Implemented specialized Flask endpoints processing JSON data payloads (`request.get_json`) and returning responses via `jsonify`.
* User & Theme Management: Supports user deletion cascaded across all tables via database cursors and manual SQL executions.
* Token-based Authorization: Uses secure, URL-safe random tokens (`secrets.token_urlsafe()`) to authenticate requests without raw credentials.
* Matchmaking Algorithm: Processes arrays of targeted themes, tracks expert profiles, filters duplicates, and returns sorted lists based on the highest theme overlap.
* Rating System: Processes incoming evaluation scores, validates single-vote policies per user, and automatically recalculates the global average score.

## Installation & Setup
1. Clone the repository.
2. Ensure Python is installed.
3. Run the server application script: `python app.py`
